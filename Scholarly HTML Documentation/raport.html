1. Introduction

1.1 Purpose

The purpose of this document is to present in detail the main product scope, its features and how it was intended to function. It could serve as a guide to new users of the product.

1.2 Product Scope

Chor (Smart Children Monitor), retitled by us as “Kidpoint”, is a Web system that monitors children in real-time. When it comes to benefits, we believe that nowadays, technology can ease the life of a parent more than ever. Our objectives are helping parents to effortlessly let their children freely explore the world and enlarging their family's horizon while staying connected!

2. Overall Description

2.1 Product Functions

Once logged in, the user can have access to their added children's location, he receives notifications, he can add new children or new people who are relevant to their children's interaction. For a more detailed user experience, see the user flow diagram below.

2.2 User Classes and Characteristics

The types of possible users on our application are:

  • Unauthenticated user

    • The user can read the "About Us" section of the appplication, can create a new account or can log in to an existing account.

  • Authenticated user
    • The user can access his children's location in real-time.
    • The user can add new children to the children list.
    • The user can add new locations to the locations list.
    • The user can set a reference point for a specific child.
    • The user receives certain notifications.

2.3 Operating Environment

The application will work on all operating systems available, such as Windows, Linux, macOS, on all web browsers and on mobile operating systems such as iOS, Android and Windows.

2.4 User Documentation

Our application will be easy to navigate on, no technological requirements will be needed for the average user of the application. If demanded, this entire documentation can serve as a user manual/guideline of the application.

4. System Features

4.1 Login

  • 4.1.1 Description and Priority
    • Any user of our application must provide their already existing username and their registered password in order to successfully log in into our application. This is a High priority feature.
  • 4.1.2 Stimulus/Response Sequences List
    • The user must enter an already existing username and their password.
    • If the username does not exist in our database, an error message will be displayed.
    • If the password is incorrect, an error message will be displayed.
    • If both the username and password are correct, the log in is successfully made and the user is officially authenticated.
  • 4.1.3 Functional Requirements
    • The username and the password that the user is trying to enter must already exist in our database.

4.2 Sign Up

  • 4.2.1 Description and Priority
    • Any non-existing user of our application must provide a not taken username, a valid e-mail address and a password in order to successfully sign up into our application. This is a High priority feature.
  • 4.2.2 Stimulus/Response Sequences List
    • The user must enter a non-existing username, the valid e-mail address and a password.
    • If the username already exists in our database, an error message will be displayed, asking the user to choose another username for their account.
    • If the e-mail address is invalid, an error message will be displayed.
    • If the e-mail address is already used by another user account on our application, an error message will be displayed.
    • If the username is not taken and the e-mail address is valid, the brand new account will be created successfully.
  • 4.2.3 Functional Requirements
    • The username and the e-mail address must not exist in our database.

4.3 My Children

  • 4.3.1 Description and Priority
    • A successfully logged in user will have access to the whole My Children Page that will contain the My Children feature. This feature will display a list of all registered children for that specific account, which includes the name of each child and their last location where they were seen on the map. This is a Medium priority feature. In this feature, the user can either add a new child to the list or delete an existing one.
  • 4.3.2 Stimulus/Response Sequences List
    • If the user wants to delete an already existing child in the list, then they should simply press the "x" button in the upper right corner of each child-card.
    • If the user wants to add a new child, he has to press the "Add Child" button, which then opens up in a new "Add Child Page". The user has to enter the first and last name of the newly added child (chosen by him) and also a unique code that they received when they bought the GPS bracelet for their child.
    • If the unique code is invalid, an error message will be displayed.
  • 4.3.3 Functional Requirements
    • The unique code must only come from one of our certified products.

4.4 Notifications Center

  • 4.4.1 Description and Priority
    • A logged in user will constantly receive notifications related to their children. The type of priority is Medium. The types of notifications are:
      • A child is distancing himself more than 15m away from their chosen reference point.
      • A child has encountered another user's child.
      • A child is getting close (less than 15m) to a location from the " My Locations" page.
      • A child is getting close (less than 15m) to / distancing from a person from the " My Friends" page.
      • A child has hit an obstacle (object, animal etc.).

    4.5 My Locations

    • 4.5.1 Description and Priority
      • The user can create his own list of locations relevant for him, constantly adding new ones. Also, for every child, he can choose a certain reference point in order to be announced if the child is leaving a specific space.
    • 4.5.2 Stimulus/Response Sequences List
      • If the user wants to see all of his locations, he has to push the "See Locations" button.
      • If the user wants to add a new location, he has to access the "My Locations" page first, by pressing the "See Locations" button. Then, he has to push the "Add Location" button. Where he will choose a valid address on the map and a name for the new location.
      • If the user wants to choose a new refence point he has to access the "My Locations" page from the relevant child's profile. Then, he has to push the "Change Reference Point" button and select a new one from the existing locations.
      • If the user introduces an invalid address when wanting to add a new location, an error message will be displayed.
    • 4.5.3 Functional Requirements
      • The new address added to the locations list must be a valid one.

    4.6 My Friends

    • 4.6.1 Description and Priority
      • The user can create his own group of other users that will be able to access all their children's data about their location.
    • 4.6.2 Stimulus/Response Sequences List
      • If the user wants to add another friend in their list, he must push the "Add Friend" button, which will then open up in the "Add Friend" page. To add a friend, the user must enter the username of the other account and give them a "role" such as Family member, Babysitter etc.
      • If the username which the user wants to add as friend does not exist, an error message will be displayed.
    • 4.6.3 Functional Requirements
      • The friend's username must already exist in our database.

    4.7 My Account

    • 4.7.1 Description and Priority
      • The user can access and modify their account's data. This is a Low priority feature.
    • 4.7.2 Stimulus/Response Sequences List
      • If the user wants to modify their username/e-mail address/password, he must simply push their "Edit" button.
      • If the e-mail address they are trying to modify is not a valid one, an error message must be displayed.
    • 4.7.3 Functional Requirements
      • The newly modified e-mail address must be a valid one.
      • The newly modified username must be a non-existing one.
